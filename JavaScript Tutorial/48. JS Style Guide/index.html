JavaScript 스타일 가이드 및 코딩 규칙

모든 자바스크립트 프로젝트에 항상 동일한 코딩 규칙을 사용하십시오.

JavaScript 코딩 규칙

코딩 규칙은 프로그래밍의 스타일 가이드 라인입니다. 그들은 일반적으로 다음을 포함합니다.

  1. 변수 및 함수의 이름 지정 및 선언 규칙
  2. 공백, 들여 쓰기 및 설명 사용 규칙
  3. 프로그래밍 실습 및 원리

코딩 규칙 안전한 품질

  1. 코드 가독성 향상
  2. 코드 유지보수가 쉬워집니다.

코딩 규칙은 팀이 따라야하는 규칙을 문서화하거나 개별 코딩 연습일 뿐입니다.

이 페이지는 W3Schools에서 사용되는 일반적인 JavaScript 코드 규칙을 설명합니다.
다음 장의 "베스트 프랙티스"를 읽고 코딩 실수를 피하는 방법도 배워야 합니다.

변수 이름

W3schools 에서는 식별자 이름(변수 및 함수)에 camelcase를 사용합니다.

모든 이름은 문자로 시작합니다.

이 페이지의 맨 아래에 이름 지정 규칙에 대한 더 자세한 설명이 나와있습니다.

    firstName = "John";
    lastName = "Doe";

    price = 19.90;
    tax = 0.20;

    fullPrice = price + (price * tax);

연산자 주변 공간

연산자 (= + - * /)와 쉼표 다음에 공백을 넣으십시오.

    var x = y + z;
    var values = ["Volvo", "Saab", "Fiat"];

코드 들여 쓰기

코드 블록의 들여 쓰기에는 항상 2칸을 사용하십시오.

    function toCelsius(fahrenheit) {
      return (5 / 9) * (fahrenheit - 32);
    }

들여쓰기에 탭을 사용하지 마십시오. 다른 편집기는 탭을 다르게 해석합니다.

정책 규정

간단한 문에 대한 일반 규칙

- 항상 간단한 문장을 세미콜론으로 끝내십시오.

    var values = ["Volvo", "Saab", "Fiat"];

    var person = {
      firstName: "John",
      lastName: "Doe",
      age: 50,
      eyeColor: "blue"
    };

복합 명령문에 대한 일반 규칙

  • 첫 번째 줄 끝부분에 여는 브라켓을 놓습니다.
  • 입구 브라켓 앞에 한칸을 사용하십시오.
  • 공백을 넣지 않고 새 줄에 닫는 괄호를 놓습니다.
  • 복잡한 명령문을 세미콜론으로 끝내지 마십시오.
    function :
    function toCelsius(fahrenheit) {
      return (5 / 9) * (fahrenheit - 32);
    }

    roop :
    for(var i = 0; i < 5; i++) {
      x += i;
    }

    conditionals :
    if (time < 20) {
      greeting = "Good day";
    } else {
      greeting = "Good evening";
    }

객체 규칙

객체 정의에 대한 일반 규칙 :

  • 객체 이름과 같은 줄에 여는 대괄호를 배치하십시오.
  • 콜론과 각 속성과 값 사이에 하나의 공백을 사용하십시오.
  • 숫자 값이 아닌 문자열 값 주위에 따옴표를 사용하십시오.
  • 마지막 속성 값 쌍 뒤에 쉼표를 추가하지 마십시오.
  • 공백을 넣지 않고 새 줄에 닫는 괄호를 놓습니다.
  • 항상 세미콜론으로 객체 정의를 종료하십시오.
    var person = {
      firstName: "John",
      lastName: "Doe",
      age: 50,
      eyeColor: "blue"
    };

짧은 객체는 다음과 같이 속성 사이의 공백만 사용하여 한 줄에 압축하여 작성할 수 있습니다.

    var person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"};

이름 지정 규칙

항상 모든 코드에 대해 동일한 명명 규칙을 사용하십시오.

  • camelCase로 작성된 변수 및 함수 이름
  • 대문자로 쓰여진 글로벌 변수 (우리는 아니지만 꽤 일반적입니다)
  • 상수 (PI와 같은)는 대문자로 쓴다.

하이픈은 빼기 시도로 오인 될 수 있습니다. JavaScript 이름에는 하이폰을 사용할 수 없습니다.

$ 기호로 이름을 시작하지 마십시오. 많은 JavaScript 라이브러리 이름과 충돌하게 됩니다.
