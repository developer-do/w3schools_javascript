JSON vs XML

JSON과 XML을 모두 사용하여 웹 서버에서 데이터를 수신 할 수 있습니다.

다음 JSON 및 XML 예제는 모두 3명의 직원을 포함하는 직원 객체를 정의합니다.

    JSON Example
    {
      "employees": [
        { "firstName" : "John", "lastName" : "Doe" },
        { "firstName" : "Anna", "lastName" : "Smith" },
        { "firstName" : "Peter", "lastName" : "Jones" }
      ]
    }

    XML Example
    
      
        John Doe
      
      
        Anna Smith
      
      
        Peter Jones
      
    
    XML은 코드로 확인해야 한다.

JSON은 XML과 비슷합니다.

  • JSON과 XML 모두 "자체설명"(사람이 읽을 수 있음)
  • JSON과 XML은 모두 계층적 (값 내 값)
  • JSON과 XML 모두 많은 프로그래밍 언어에서 파싱하고 사용할 수 있습니다.
  • JSON과 XML 모두 XMLHttpRequest를 사용하여 가져올 수 있습니다.

JSON은 XML과 다르기 때문에

  • JSON은 종료 태그를 사용하지 않습니다.
  • JSON이 더 짧습니다.
  • JSON은 읽고 쓰는 것이 더 빠릅니다.
  • JSON은 배열을 사용할 수 있습니다.

가장 큰 차으점은 다음과 같습니다.

XML은 XML 구문 분석기로 구문 분석되어야 합니다. JSON은 표준 JavaScript 함수로 파싱 할 수 있습니다.

JSON이 XML보다 나은이유

XML은 JSON보다 구문 분석하기가 훨씬 어렵다.
JSON은 바로 사용할 수 있는 JavaScript 객체로 파싱됩니다.

AJAX응용 프로그램의 경우 JSON은 XML보다 빠르고 쉽습니다.

XML 사용

  • XML 문서 가져오기
  • XML DOM을 사용하여 문서를 순환합니다.
  • 값 추출 및 변수 저장

JSON 사용

  • JSON 문자열 가져오기
  • JSON.Parse JSON문자열
