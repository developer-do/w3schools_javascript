자동 HTML 양식 유효성 검사

양식 필드 (fname)가 비어 있으면 필수 속성이 이 양식을 제출하지 못하게 합니다.

    HTML 양식 예제
    form action ="/action_page.php" method="post"
      input type="text" name="fname" required

    /form

데이터 유효성 검사

데이터 유효성 검사는 사용자 입력이 깨끗하고 정확하며 유용하다는 것을 보증하는 프로세스 입니다.

일반적인 유효성 검사 작업은 다음과 같습니다.

  • 사용자가 모든 필수 필드를 채우고 있습니까?
  • 사용자가 유효한 날짜를 입력 햇습니까?
  • 사용자가 숫자 필드에 텍스트를 입력 했습니까?

대부분의 경우 데이터 유효성 검사의 목적은 올바른 사용자 입력을 보장하는 것입니다.

유효성 검사는 여러 가지 방법으로 정의할 수 있으며 디앵힌 빙밥으로 베포할 수 있습니다.

서버 측 유효성 검증은 입력이 서버로 송신 된 후 웹 서버에 의해 수행됩니다.

클라이언트 측 유횽성 검사는 입력이 웹서버로 전송되기 전에 웹 브라우저에 의해 수행됩니다.

HTML 제약 조건 검증

HTML5는 제약 조건 유효성 검사 라는 새로운 HTML 유효성 검사 개념을 도입했습니다.

HTML 제약 조건 유효성 검사는 다음을 기반으로 합니다.

  • 제약 조건 유효성 검사 HTML 입력 속성
  • 제약 조건 검증 CSS Pseudo Selectors
  • 제약 조건 유효성 검사 DOM 속성 및 메서드

제약 조건 유효성 검사 HTML 입력 속성

Attribute Description
disabled 입력 요소를 사용하지 않도록 지정
max 입력 요소의 최대값을 지정함
min 입력 요소의 최소값을 지정함
pattern 입력 요소의 값 패턴을 지정함
required 입력 필드에 요소가 필요하도록 지정
type 입력 요소의 유형을 지정함

제약 조건 검증 CSS 의사 선택기

Selector Description
:disabled "사용 안 함" 특성이 지정된 입력 요소 선택
:invalid 값이 유효하지 않은 입력 요소 선택
:optional "필수" 특성이 지정되지 않은 입력 요소 선택
:required "필수" 특성이 지정된 입력 요소 선택
:valid 유효한 값이 있는 입력 요소 선택
