JavaScript 코드 속도를 높이는 방법.

roop에서 활동 감소

루프는 종종 프로그래밍에 사용됩니다.

for문을 포함하여 루프의 각 명령문은 루프의 각 반복마다 실행됩니다.

루프 외부에 배치 할 수 있는 명령문 또는 지정은 루프를 더 빠르게 실행합니다.

    나쁜:

    var i;
    for(i = 0; i < arr.length; i++){

    }

    더 나은 코드:

    var i;
    var l = arr.length;
    for(i = 0; i < l; i++){

    }

나쁜 코드는 루프가 반복될때마다 배열의 length속성에 접근합니다.

더 나은 코드는 루프 외부의 length 속성에 접근하여 루프를 빠르게 실행합니다.

DOM 접근 줄이기

HTML DOM 접근은 다른 JavaScript문과 비교할 때 매우 느립니다.

DOM 요소에 여러번 접근 하려는 경우 한 번 접근하여 로컬변수로 사용하십시오.

    var obj;
    obj = document.getElementById("demo");
    obj.innerHTML = "Hello";

DOM 크기 줄이기

HTML DOM의 요소 수를 작게 유지하십시오.

이렇게하면 항상 페이지 로드가 향상되고 특히 작은 장치에서 렌더링(페이지 표시)이 빨라집니다.

getElementsByTagName과 같은 DOM 검색을 시도 할 때마다 더 작은 DOM의 이점이 있습니다.

불필요한 변수 피하기

값을 저장하지 않으려는 경우 새 변수를 만들지 마십시오.

다음과 같이 코드를 대체 할 수 있습니다.

    var fullName = firstName + " " + lastName;
    document.getElementById("demo").innerHTML = fullName;

    위의 코드를 이것으로:
    document.getElementById("demo").innerHTML = firstName + " " + lastName;

JavaScript 로드 지연

스크립트 페이지 본문의 맨 아래에 부면 브라우저가 페이지를 먼저 로드 할 수 있습니다.

스크립트가 다운로드 되는 동안 브라우저는 다른 다운로드를 시작하지 않습니다. 또한 모든 구문 분석 및 렌더링 작업이 차단 될 수 있습니다.

HTTP 사양은 브라우저가 두 개 이상의 구성 요소를 동시에 다운로드 해서는 안된다는 것을 정의합니다.

대안은 스크립트 태그에 reqils="ture"를 사용하는 것이다. 지연 속성은 페이지 파싱이 완료 된 후 스크립트를 실행해야 하지만 외부 스크립트에서만 작동한다고 지정합니다.

가능 한 경우 페이지가 로드 된 후 코드별로 스크립트를 페이지에 추가 할 수 있습니다.

    window.onload = function(){
      var element = document.createELement("script");
      element.src = "myScript.js";
      document.body.appendChild(element);
    };

with 사용하지 않기

with 키워드를 사용하지 마십시오. 속도에 부정적인 영향을 미칩니다. 또한 JavaScript 범위를 혼란스럽게 합니다.

"use strict"모드에서는 with 키워드를 사용할 수 없습니다.
