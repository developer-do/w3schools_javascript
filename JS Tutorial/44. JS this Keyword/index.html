what is this?

JavaScript this keyword는 속한 객체를 나타낸다.

이 값은 사용되는 위치에 따라 다른 값을 가집니다.

  • method에서 this는 소유자 객체를 참조합니다.
  • 혼자 있는 this이면 전역 객체(window)를 참조합니다.
  • function에서 this는 전역 객체(window)를 참조합니다.
  • "use strict"에서 function의 this는 undefined입니다.
  • 이벤트 안에서, this는 이벤트를 받은 요소(element, 객체)를 가리킵니다.
  • call() 및 apply()와 같은 메소드는 this를 어떤 element이든 참조할 수 있습니다.

this가 method에서 하는 것

object method에서 this는 method의 "owner"를 참조합니다.

이 페이지의 상단의 예에서, this person 객체를 나타냅니다.

person 객체는 fullName method의 "owner"입니다.

    fullName : function() {
      return this.firstName + " " + this.lastName;
    }

this alone

this가 단독으로 사용되는 경우, 소유자는 전역객체 이기 때문에 this 는 global 객체를 참조합니다.

브라우저 창에서 전역 객체는 [object window]입니다.

    var x = this;

"use strict"에서는 this가 단독으로 사용되는 경우, this또한 window 객체를 가르킵니다. [object window]

    "use strict";
    var x = this;

this property (default)

자바스크립트 함수에서 함수 소유자는 이름 위한 기본 바인딩이다.

즉, 함수에서 이것은 global obejct[obejct window]를 나타낸다.

    function myFunction(){
      return this;
    }

this property ("use strict")

자바스크립트 "use strict"는 기본 바인딩을 허용하지 않습니다.

함수에서 사용할 때, 엄경한 모드에서 this는 undefined입니다.

    "use strict";
    function myFunction(){
      return this;
    }

event handler의 this

HTML 이벤트 핸들러에서 이는 이벤트를 수신 한 HTML 요소를 나타냅니다.

    button onclick="this.style.display='none'"
      Click to Remove Me !
    button

객체 method 바인딩

이 예에서 이것은 개인 객체(person obejct는 함수의 "소유자" 임)

    var person = {
      firstName : "John",
      lastName  : "Doe",
      id        : 5566,
      myFunction: function(){
        return this;
      }
    };

명시적 함수 바인딩

call()과 apply() methods는 미리 정의된 JavaScript 방법입니다.

이 두가지 방법 모두 다른 객체 method를 인수로 호출하는 데 사용할 수 있다.

이 예에서 person1.fullName에서 person2를 인수로 호출할 때, person1의 방법일지라도 person2를 나타낸다.

    var person2 = {
      firstName: "John",
      lastName: "Doe",
    }
    var person3 = {
      firstName: "Kim",
      lastName: "DoHyun"
    }

    console.log(person1.fullName.call(person2));
    console.log(person1.fullName.call(person3));
